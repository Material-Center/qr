# 手机号注册创建任务 OpenAPI 接入说明

版本：v1  
适用场景：第三方系统通过 OpenAPI 查询当前可用设备数量，并创建手机号注册任务。

## 基础信息

接口基础地址：

```text
http://210.16.170.132:1111/api
```

所有请求和响应均使用 JSON。

## 认证方式

调用接口时需要携带 OpenAPI Token。Token 由平台提供。

推荐使用请求头：

```http
X-Open-Api-Token: <your_openapi_token>
```

也支持 Bearer 方式：

```http
Authorization: Bearer <your_openapi_token>
```

## 通用响应格式

成功响应：

```json
{
  "code": 0,
  "data": {},
  "msg": "成功"
}
```

失败响应：

```json
{
  "code": 7,
  "data": {},
  "msg": "错误原因"
}
```

认证失败时 HTTP 状态码为 `401`，响应示例：

```json
{
  "code": 7,
  "data": null,
  "msg": "OpenAPI token无效"
}
```

## 查询可用设备

用于查询当前在线设备和空闲设备数量。建议在创建任务前先调用该接口，确认有空闲设备可处理任务。

请求方式：

```http
GET /phoneRegisterTask/open-api/promoter/device-stats
```

完整地址：

```text
GET http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/device-stats
```

请求参数：无

响应示例：

```json
{
  "code": 0,
  "data": {
    "deviceOnlineCount": 3,
    "deviceIdleCount": 2
  },
  "msg": "获取成功"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| deviceOnlineCount | number | 当前在线设备数量 |
| deviceIdleCount | number | 当前空闲设备数量 |

curl 示例：

```bash
curl -X GET "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/device-stats" \
  -H "X-Open-Api-Token: <your_openapi_token>"
```

## 创建手机号注册任务

用于创建一个手机号注册任务。任务创建后，平台设备会按任务状态自动处理。

请求方式：

```http
POST /phoneRegisterTask/open-api/promoter/task
```

完整地址：

```text
POST http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/task
```

请求头：

```http
Content-Type: application/json
X-Open-Api-Token: <your_openapi_token>
```

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| phone | string | 是 | 手机号，必须为 11 位数字 |
| startDelaySeconds | number | 否 | 服务端延迟开始秒数，默认 `0`；取值范围 `0` 到 `600`。大于 `0` 时，任务会先创建为 `pending`，到达 `availableAt` 后才允许设备领取 |
| reserveDevice | boolean | 否 | 是否尝试预占空闲设备，默认 `false`。建议仅在 `startDelaySeconds > 0` 时传 `true`；如果创建时有空闲设备，平台会立即预占一台设备，延迟期间该设备不再接其他任务；如果没有空闲设备，仍会创建未锁设备的延迟任务 |

请求示例：

```json
{
  "phone": "18878309701"
}
```

延迟创建并尝试预占设备示例：

```json
{
  "phone": "18878309701",
  "startDelaySeconds": 30,
  "reserveDevice": true
}
```

响应示例：

```json
{
  "code": 0,
  "data": {
    "id": 123,
    "createdAt": "2026-05-26T10:00:00+08:00",
    "phone": "18878309701",
    "smsReceiveMode": "USER_SENT_TO_TX",
    "status": "pending",
    "statusCode": null,
    "lastError": "",
    "needPromoterCode": false,
    "holderDeviceId": "device-001",
    "availableAt": "2026-05-26T10:00:30+08:00",
    "expiresAt": "2026-05-26T10:30:30+08:00",
    "finishedAt": null
  },
  "msg": "创建成功"
}
```

响应字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 任务 ID |
| phone | string | 任务手机号 |
| smsReceiveMode | string | 收码方式，当前固定为 `USER_SENT_TO_TX` |
| status | string | 任务状态 |
| statusCode | number/null | 状态码，失败或设备上报时可能返回 |
| lastError | string | 最近一次错误信息 |
| needPromoterCode | boolean | 是否需要提交验证码 |
| holderDeviceId | string/null | 当前持有或预占任务的设备 ID。未预占或未领取时可能为空 |
| availableAt | string/null | 任务可被设备领取的时间。不传 `startDelaySeconds` 或为 `0` 时通常为空 |
| expiresAt | string | 任务过期时间 |
| finishedAt | string/null | 任务完成时间，未完成时为 null |

curl 示例：

```bash
curl -X POST "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/task" \
  -H "Content-Type: application/json" \
  -H "X-Open-Api-Token: <your_openapi_token>" \
  -d '{"phone":"18878309701"}'
```

延迟创建 curl 示例：

```bash
curl -X POST "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/task" \
  -H "Content-Type: application/json" \
  -H "X-Open-Api-Token: <your_openapi_token>" \
  -d '{"phone":"18878309701","startDelaySeconds":30,"reserveDevice":true}'
```

## 创建收码任务

用于创建需要外部提交验证码的手机号注册任务。任务创建后，设备进入验证码阶段时，接口返回的 `needPromoterCode` 会变为 `true`，调用方需要获取验证码并提交。

请求方式：

```http
POST /phoneRegisterTask/open-api/promoter/receive-task
```

请求参数同“创建手机号注册任务”，但收码方式固定为 `PLATFORM_SEND`。

请求示例：

```json
{
  "phone": "18878309701"
}
```

curl 示例：

```bash
curl -X POST "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/receive-task" \
  -H "Content-Type: application/json" \
  -H "X-Open-Api-Token: <your_openapi_token>" \
  -d '{"phone":"18878309701"}'
```

## 查询任务状态

用于查询当前 Token 用户自己创建的任务。程序重启后可用本接口恢复已创建任务的状态，避免重复创建。

请求方式：

```http
GET /phoneRegisterTask/open-api/promoter/task/{taskId}
```

curl 示例：

```bash
curl -X GET "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/task/123" \
  -H "X-Open-Api-Token: <your_openapi_token>"
```

## 提交验证码

仅收码任务进入 `needPromoterCode=true` 后调用。

请求方式：

```http
POST /phoneRegisterTask/open-api/promoter/submit-code
```

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| taskId | number | 是 | 任务 ID |
| verifyCode | string | 是 | 验证码 |

请求示例：

```json
{
  "taskId": 123,
  "verifyCode": "123456"
}
```

curl 示例：

```bash
curl -X POST "http://210.16.170.132:1111/api/phoneRegisterTask/open-api/promoter/submit-code" \
  -H "Content-Type: application/json" \
  -H "X-Open-Api-Token: <your_openapi_token>" \
  -d '{"taskId":123,"verifyCode":"123456"}'
```

## 文件导入收码工具

`phonecodeworker` 支持从文件导入手机号并自动创建收码任务。文件第一行是验证码 API，后续每行一个手机号：

```text
https://code.example/api?phone=
18878309701
18878309702
```

运行示例：

```bash
cd phonecodeworker
go run . -token "<your_openapi_token>" -input "/Users/fupeng/Downloads/500 (4).txt"
```

工具会把状态保存到 `<input>.state.json`。程序退出后再次启动会优先恢复已创建但未完成的任务。

## 推荐调用流程

1. 调用“查询可用设备”接口。
2. 当 `deviceIdleCount > 1` 时，获取手机号。
3. 自己发码场景调用“创建手机号注册任务”；收码场景调用“创建收码任务”。
4. 收码任务创建后，轮询“查询任务状态”；当 `needPromoterCode=true` 时调用验证码 API。
5. 获取验证码后调用“提交验证码”。
6. 创建或提交失败时，根据 `msg` 判断原因，并稍后重试。

## 常见错误

| msg | 说明 |
| --- | --- |
| 缺少OpenAPI token | 未传 Token |
| OpenAPI token无效 | Token 格式错误或签名无效 |
| OpenAPI token不存在或已作废 | Token 未启用或已被作废 |
| OpenAPI token已过期 | Token 已超过有效期 |
| 手机号不能为空 | 请求体缺少 phone |
| 手机号必须为11位数字 | 手机号格式不正确 |
| 手机号注册已关闭 | 平台暂时关闭了手机号注册任务 |
| 当前账号已禁用任务创建 | 当前 Token 对应账号被禁用创建任务 |
| 该手机号段暂不支持提交 | 手机号号段被限制 |
| 任务不存在 | 任务 ID 不存在，或不属于当前 Token 用户 |
| 当前任务未处于待地推验证码状态 | 收码任务尚未进入可提交验证码阶段 |
| 当前任务收码方式不支持提交验证码 | 非收码任务调用了提交验证码接口 |
| startDelaySeconds不能小于0 | 延迟秒数不能为负数 |
| startDelaySeconds不能超过600 | 延迟秒数超过服务端限制 |

## 注意事项

- Token 应妥善保存，不要暴露在客户端页面或公开代码仓库中。
- 创建任务前建议先判断空闲设备数量，避免无设备可处理时持续提交任务。
- `startDelaySeconds` 不传或传 `0` 时保持旧行为，任务创建后立即可被设备领取。
- `reserveDevice=true` 且有空闲设备时，平台会把设备计为 busy，因此查询可用设备接口中的 `deviceIdleCount` 会减少，`deviceOnlineCount` 不变。
- `reserveDevice=true` 但创建时没有空闲设备时，接口不会因为无法预占而失败；任务仍会创建，`holderDeviceId` 为空，到达 `availableAt` 后由后续空闲设备领取。
- 预占只提供 `availableAt` 到达后的一段优先领取窗口；如果预占设备未按时领取，平台会释放预占并允许其他设备领取。
- 同一个手机号不建议重复提交。
- 接口调用间隔建议不低于 3 秒。
