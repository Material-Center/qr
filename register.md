# 注册任务功能说明

账号是地推进行注册的，团长和管理员没有这个权限，只能看数据统计，我们系统是对地推的任务做管理的。
1. 管理员可以看到各个团长/地推，注册账号的成功数/失败数组/正在注册数，团长的就是下面地推的汇总
2. 管理员和团长，可以看某个地推的任务详细记录，上面需要展示任务总数（成功/失败/处理中）
3. 地推有个自己的任务记录页面，查看自己做过的任务，上面需要展示任务总数（成功/失败/处理中）
4. 团长和地推看到的任务记录手机号都是要mark的，只能看前三位和后三位
3. 地推需要有个创建任务的页面，这个页面是需要兼容h5使用的，方便在手机上使用

## 地推注册账号（创建任务）流程

具体实现参考go项目 /Users/fupeng/Workspace/github/Material-Center/qpi/tool
项目tool目录下有几个工具函数，对应下面的流程的
1. QueryPhoneBindQQ 查看手机号绑定的qq
2. ChangePassword 更新对应qq的密码
3. QQPasswordLogin 登录账号，并提取缓存
4. 查达人（还未实现是一个http调用）
上面的项目是一个私有的go mod，托管在github上的

这里本质上地推就是触发上面的代码流程，具体实现细节后面看这个qpi的go lib看具体流程，界面操作如下

1. 有个输入框，先输入手机号，接受一个验证码进入下一步
2. 手机号有绑定qq账号的话，默认用第一个账号，展示结果找到qq号，没有qq账号的话提示没有可用qq账号，任务失败并记录原因
3. 有绑定qq账号的话，提示正在改密，改密流程还需要接受一个验证码，后看成功还是失败。
4. 改密成功的话，触发登录流程，还需要收一个验证码，填入后触发登录流程和查达人，保存任务结果

上面流程就是第一步填入手机号，后面需要接受三个验证码，这里的每一步都可能存在失败的情况，失败情况地推可以选择
重试或者是失败（这个可以在服务端控制，比如某些错误可以重试，某些不能重试），每个地推一次只能完成一个任务，如果任务还没完成，再次进入道这个创建页面，展示未完成任务，完成任务后回到第一步流程，一个手机号输入框
任务整体有个超时时间（10min），超过系统默认改成已完成

这里任务表需要记录几个数据
1. 手机号
2. qq账号
3. 密码
4. 登录缓存数据，是一个ini的配置文件
5. 是否达人
6. 对应任务的状态 （这里是一个状态码 > 0 业务失败 < 0 系统失败 == 0 注册成功）
7. 地推账号id、
8. 改密时间/登录时间 保留两个时间字段
9. 任务完成时间，有这个代表任务完成了

注意这里任务实现上采用客户端轮训的方案比较简单，服务端控制前端页面展示状态，这里在轮训过程中，应该使用redis缓存来避免系统db压力
